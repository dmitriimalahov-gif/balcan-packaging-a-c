# Домен: упаковка и каталог макетов

## Контекст

Этот репозиторий покрывает прежде всего **каталог макетов** (строки синхронизированы с Excel по `excel_row`), **печать** (слоты на листе, SVG/PDF), **CG** (ножи, цены, привязка позиций), **cutii** (помесячные объёмы и подтверждения сопоставлений), **остатки по GMP** и вспомогательные расчёты.

Подробная таблица сущностей и экранов: [`architecture/architecture_domain_inventory.md`](../architecture_domain_inventory.md).

## Соответствие целевому ERP (bootstrap)

| Сейчас (a-c) | Целевой модуль bootstrap |
|----------------|---------------------------|
| `packaging_items` + PDF | `packaging_catalog`, `artwork_and_files` |
| Печать / лист | `production_planning` (часть), документы |
| CG knives/prices | `engineering` + внешний контракт типографии |
| `stock_on_hand` | `warehouse` (упрощённо) |
| Планировщик Excel | `planning` / `costing` (частично) |

## Ключевые идентификаторы

- **`excel_row`** — номер строки в мастер-Excel; в Postgres добавлен стабильный **`id`** у позиции каталога.
- **`gmp_code`** — логический SKU-якорь для остатков и отчётов.

## Инварианты сценариев

- Импорт макетов: валидация через [`packaging_schemas`](../../packaging_schemas/) до записи в БД.
- Раскладка на лист: использовать общие функции из [`packaging_print_planning`](../../packaging_print_planning.py), не копировать формулы в UI.

## Расширение

Новые сущности (`PackagingProduct`, заказы, BOM) вводятся через отдельные миграции и модули; связь с `excel_row`/`gmp_code` — через явные FK или сопоставляющие таблицы (см. ADR-0001).
