# Роли AI в одном чате Cursor (как в ERP Bootstrap §18)

## Важно понимать технически

В Cursor **одно окно чата = одна языковая модель**. Отдельные фоновые «боты», которые сами параллелят задачи без вашего сообщения, из коробки не подключаются репозиторием.

**«Много агентов» у нас означает:**

1. **Оркестратор** ([`.cursor/rules/orchestrator.mdc`](../.cursor/rules/orchestrator.mdc), `alwaysApply: true`) — в ответе явно выбирается **роль** и её ответственность.
2. **Правила ролей** ([`.cursor/rules/role-*.mdc`](../.cursor/rules/)) — подключаются **автоматически по файлам** (globs), с которыми вы работаете, и по **описанию** правила.
3. **Общие правила** (`core.mdc`, `packaging_viewer.mdc`, …) — остаются для конкретных областей кода.

Итог: вы ведёте **один диалог**, а ответы становятся **специализированными** за счёт ролей и контекста файлов.

## Как явно вызвать роль

В начале сообщения укажите префикс (удобно копировать):

| Префикс | Роль | Файлы правил |
|-----------|------|----------------|
| `[Архитектор]` | Границы модулей, ADR | `role-architect.mdc` |
| `[Домен]` | Смысл упаковки, макеты, CG | `role-domain.mdc` |
| `[Бэкенд]` | API, сервисы, модули, worker | `role-backend.mdc` |
| `[Фронт]` | React SPA | `role-frontend.mdc` |
| `[SQL]` | Postgres, Alembic, SQLite, витрины | `role-data-sql.mdc` |
| `[Планирование]` | Раскладка, прогноз | `role-planning.mdc` |
| `[QA]` | Тесты, регрессия | `role-qa.mdc` |
| `[Доки]` | Документация, схемы | `role-docs.mdc` |

Пример:

```text
[Бэкенд] Добавь эндпоинт GET /api/v1/items/{excel_row} только для чтения.
```

Модель должна начать ответ со строки вида **`Роль: Бэкенд`**.

## Как это сочетается с @-упоминаниями

- Прикрепляйте **`AGENTS.md`** к важным запросам — там краткий контракт репозитория.
- Для архитектуры — файлы из `docs/architecture/` или `docs/adr/`.
- Cursor подтянет **дополнительные** `role-*.mdc`, если открыты или упомянуты соответствующие пути.

## Полный эталон процесса

Разделы **§18–§20** в [`docs/reference/CURSOR_ERP_BOOTSTRAP.md`](reference/CURSOR_ERP_BOOTSTRAP.md) и корневой [`AGENTS.md`](../AGENTS.md).

## Ограничения

- **Параллельные** независимые агенты в одном чате без очереди сообщений Cursor не эмулирует: для параллельной работы используйте отдельные чаты или встроенные функции продукта (если включены в вашей подписке).
- Роли **не заменяют** code review человеком на критичных изменениях схемы БД и продакшена.
